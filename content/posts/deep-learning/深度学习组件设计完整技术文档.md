## 1. Dropout 设计

### 1.1 放置位置

- **全连接层之后**：主要位置，常用丢弃率 0.5。
- **卷积层之后**：使用变体 `SpatialDropout2D`，丢弃率 0.1~0.3。
- **循环层之后**：RNN/LSTM 使用 `Dropout`（垂直连接）和 `RecurrentDropout`（时间步间）。
- **Transformer**：每个子层（注意力、FFN）后加 Dropout，比率 0.1。

**典型顺序**：`Conv/Linear → BN → Activation → Dropout`

### 1.2 丢弃率设计

| 层类型 | 推荐丢弃率 | 说明 |
| :--- | :--- | :--- |
| 全连接层 | 0.5 | 经典值，防止过拟合 |
| 卷积层 | 0.1~0.3 | 参数共享，不宜过大 |
| 输入层 | ≤0.2 | 过大丢失原始信息 |
| 小数据集 | 降低至 0.1~0.3 | 或仅在最后一层使用 |
| 超大模型 | 0.1 或不用 | 避免欠拟合 |

### 1.3 变体选择

- **标准 Dropout**：随机丢弃神经元，用于全连接层。
- **SpatialDropout1D/2D/3D**：整张特征图或整条通道丢弃，**强烈推荐用于 CNN**。
- **AlphaDropout**：配合 SELU 激活，保持自归一化属性。

### 1.4 训练与测试行为

- **训练时**：随机丢弃，输出按 `1/(1-rate)` 缩放。
- **测试时**：不随机丢弃，自动乘以保留概率（框架自动处理）。

### 1.5 代码示例

```python
# PyTorch
self.dropout = nn.Dropout(0.5)
self.spatial_dropout = nn.Dropout2d(0.2)   # 2D spatial dropout

# Keras
layers.Dropout(0.5)
layers.SpatialDropout2D(0.2)
```

## 2. 激活函数设计

### 2.1 放置位置

**标准顺序**：`Linear/Conv → (BN) → Activation → (Dropout)`

- 隐藏层：激活函数紧跟线性变换（及可选的归一化）之后。
- 输出层：根据任务选择，通常不加激活（回归）或使用 Softmax/Sigmoid。
- Transformer FFN：`Linear1 → GELU → Linear2`。
- 残差块：通常 `Conv → BN → ReLU → Conv → BN → Add → ReLU`（最后的 ReLU 在相加后）。

### 2.2 激活函数选择

#### 隐藏层常用激活

| 激活函数 | 特点 | 推荐场景 |
| :--- | :--- | :--- |
| **ReLU** | 计算快，无正饱和，有死亡问题 | 默认首选（CNN、MLP） |
| **LeakyReLU** | 负区小斜率，解决死亡 ReLU | 发现大量神经元输出 0 时替换 |
| **ELU / SELU** | 负区饱和，均值近零；SELU 可自归一化 | 深度网络，配合 lecun_normal 初始化 |
| **Swish / SiLU** | 平滑非单调，性能有时优于 ReLU | 大型模型（EfficientNet、Transformer） |
| **GELU** | Transformer 标准 | Transformer、BERT |
| **Tanh** | 输出 [-1,1]，零中心，易饱和 | RNN 内部、生成器输出层 |
| **Sigmoid** | 输出 (0,1)，易饱和 | 仅用于二分类输出层 |

#### 输出层选择

| 任务 | 激活函数 | 说明 |
| :--- | :--- | :--- |
| 二分类 | Sigmoid | 输出概率 |
| 多分类（互斥） | Softmax | 概率和为 1 |
| 多标签分类 | Sigmoid（每个节点独立） | 每个标签独立二分类 |
| 回归（无约束） | 线性（无激活） | 直接输出实数 |
| 回归（正值） | ReLU 或 Softplus | 强制非负 |
| 回归（[-1,1]） | Tanh | 如 GAN 生成器 |

### 2.3 高级技巧

- **与 BatchNorm 顺序**：`Conv → BN → ReLU` 是主流，BN 在激活前效果更好。
- **死亡 ReLU 监控**：若大量神经元输出 0，改用 LeakyReLU 或 ELU。
- **初始化配合**：ReLU 配合 He 初始化；SELU 配合 lecun_normal。
- **输出层无激活时**：回归任务直接线性输出；若需正数，用 ReLU/Softplus。

## 3. 归一化层（Norm）设计

### 3.1 类型与适用场景

| 类型 | 归一化维度 | 适用场景 | 关键点 |
| :--- | :--- | :--- | :--- |
| **BatchNorm (BN)** | N, C, H, W | CNN，batchsize≥8 | 依赖 batch，训练/测试行为不同 |
| **LayerNorm (LN)** | C, H, W 或特征维 | RNN、Transformer、小 batch | 与 batch 无关，序列长度鲁棒 |
| **InstanceNorm (IN)** | C, H, W（每个样本独立） | 风格迁移、GAN | 保留纹理对比度 |
| **GroupNorm (GN)** | 分组通道 | 小 batchsize（1~8） | 折中方案，目标检测常用 |
| **AdaptiveNorm** | 条件自适应 | 风格迁移、域自适应 | 不常用 |

### 3.2 放置位置

#### CNN 标准顺序（推荐）
```
Conv → BN → Activation → Dropout
```
- BN 放在激活之前，因为激活函数会破坏分布的对称性。
- 训练/测试切换：必须调用 `model.eval()` 使 BN 使用滑动平均统计。

#### Transformer 中的位置

| 顺序 | 结构 | 特点 |
| :--- | :--- | :--- |
| **Post-LN**（原始） | `子层 → 残差 → LN` | 训练不稳定，需要学习率预热 |
| **Pre-LN**（改进） | `LN → 子层 → 残差` | 训练稳定，无需预热，**推荐** |

Pre-LN 示例：
```python
def forward(self, x):
    x = x + self.attn(self.norm1(x))
    x = x + self.ffn(self.norm2(x))
    return x
```

#### 残差块中的变体

- 标准 ResNet：`Conv → BN → ReLU → Conv → BN → Add → ReLU`
- 预激活 ResNet：`BN → ReLU → Conv → BN → ReLU → Conv → Add`（更适合极深网络）

### 3.3 参数设计

- **BN**：`eps=1e-5`, `momentum=0.1`。小 batchsize（<8）时效果差，改用 GN 或 LN。
- **LN**：`eps=1e-5`, `elementwise_affine=True`。对最后一维（特征维）归一化。
- **GN**：分组数常取 32 或 `channels // 4`。batchsize=1 时工作良好。
- **IN**：每个样本每个通道独立，无 momentum 参数。

### 3.4 常见陷阱

| 问题 | 现象 | 解决 |
| :--- | :--- | :--- |
| 忘记 `model.eval()` | 测试效果极差 | 验证/测试前切换 |
| 小 batchsize 用 BN | 训练震荡，收敛慢 | 改用 GN/LN 或增大 batch |
| BN 与 Dropout 顺序错误 | 训练不稳定 | 确保 BN 在前，Dropout 在后 |
| RNN 中用 BN | 失效 | 改用 LN |
| 分布式训练 BN 不同步 | 统计量不全局 | 使用 SyncBN |

## 4. 权重初始化

| 初始化方法 | 适用激活函数 | 方差公式 | 建议 |
| :--- | :--- | :--- | :--- |
| **Xavier/Glorot** | Sigmoid, Tanh, 线性 | `2/(fan_in+fan_out)` | 浅层或饱和激活 |
| **He/Kaiming** | ReLU, LeakyReLU, PReLU | `2/fan_in` | **ReLU 系列默认** |
| **正交初始化** | RNN/LSTM | 随机正交矩阵 | 缓解梯度消失/爆炸 |
| **偏置初始化** | 任意 | 通常 0 | 除 ReLU 负斜率等情况 |

**实践经验**：
- 避免全零初始化（对称破坏）。
- SELU 必须配合 `lecun_normal` 初始化。
- 残差网络最后一层卷积初始化为 0，使初始恒等映射。

## 5. 优化器与学习率调度

### 5.1 优化器选择

| 优化器 | 特点 | 适用场景 | 推荐参数 |
| :--- | :--- | :--- | :--- |
| **SGD + Momentum** | 泛化好，需调参 | CNN、小数据集 | lr=0.01~0.1, momentum=0.9, wd=1e-4 |
| **Adam** | 自适应学习率，收敛快 | Transformer、GAN、RNN | lr=3e-4~1e-3, betas=(0.9,0.999) |
| **AdamW** | 修正权重衰减 | 大部分现代模型 | 同 Adam，**推荐替代 Adam** |
| **RMSprop** | 处理非平稳目标 | 强化学习 | lr=0.001, alpha=0.99 |
| **LAMB** | 大 batch 训练 | BERT 预训练 | batchsize>64K |

### 5.2 学习率调度策略

- **阶梯下降**：每 N 个 epoch 乘以 0.1（经典 CNN）。
- **余弦退火**：余弦衰减到 0，可重启（SGDR）。
- **One Cycle**：先升后降，适合 CNN，收敛快。
- **线性预热**：前几个 epoch 从小 lr 线性升至目标 lr（Transformer 必备）。

**实践组合**：`AdamW + 余弦退火 + 线性预热` 覆盖大多数任务。

### 5.3 其他技巧

- 梯度裁剪：`max_norm=1.0`（RNN/Transformer 必备）。
- 若损失不收敛，降低 lr 到 1e-4 再逐步增加。
- 学习率寻找（LR Finder）：逐步增加 lr，选择损失下降最快的 lr。

## 6. 网络架构设计经验

### 6.1 经典设计模式

| 模式 | 核心思想 | 应用 |
| :--- | :--- | :--- |
| **残差连接** | 学习恒等映射 | 深度网络必备（>20 层） |
| **瓶颈结构** | 1x1 降维 → 3x3 → 1x1 升维 | ResNet-50, MobileNet |
| **深度可分离卷积** | 逐通道 + 逐点卷积 | MobileNet, EfficientNet |
| **Inception** | 多尺度卷积核并行 | 捕捉不同感受野 |
| **注意力机制** | 自注意力、通道/空间注意力 | Transformer, SENet, CBAM |
| **跳级连接** | 编码器-解码器跨层连接 | U-Net, FPN |

### 6.2 深度与宽度选择

- 增加深度比增加宽度通常更有效，但超过阈值收益递减（如 ResNet-152 vs 101）。
- 先设计足够深的网络确保过拟合，再用正则化减轻。
- 输入尺寸：CNN 输入为 32 或 64 的倍数（方便下采样）。

## 7. 正则化技术

| 方法 | 作用 | 经验值 |
| :--- | :--- | :--- |
| **L2 正则化（Weight Decay）** | 惩罚大权重 | CNN: 1e-4；Transformer: 0.01~0.1 |
| **L1 正则化** | 稀疏权重 | 很少单独用，可结合 L2 |
| **Dropout** | 随机丢弃神经元 | 全连接 0.5，卷积 0.1~0.3 |
| **DropConnect** | 随机丢弃连接 | 更激进，不常用 |
| **Stochastic Depth** | 随机跳过残差块 | 训练时使用，测试时全用 |
| **Label Smoothing** | 软化 one-hot 标签 | 平滑因子 0.1 |
| **MixUp / CutMix** | 混合样本和标签 | alpha=0.2~0.4 |
| **早停法** | 验证集不提升即停 | patience=5~20 epoch |

**组合建议**：
- 小数据集：Dropout(0.5) + L2(1e-4) + 早停
- 大数据集：Label Smoothing + 轻量 Dropout(0.1) + 权重衰减

## 8. 训练与调试技巧

### 8.1 过拟合/欠拟合诊断

- **过拟合**：训练损失远小于验证损失 → 增加正则化、Dropout、数据增强。
- **欠拟合**：训练损失不下降 → 增大模型、减小正则化、检查学习率/初始化。

### 8.2 关键监控指标

- 训练损失、验证损失、学习率、梯度范数、权重直方图。
- 梯度范数应在 0.1~10 之间；太大需梯度裁剪，太小需换激活/初始化。

### 8.3 训练策略

- **课程学习**：先易后难样本（如先小尺寸图像）。
- **迁移学习**：使用预训练模型，微调顶层。
- **冻结部分层**：先冻结骨干训练新头部，再解冻低 lr 微调全部。
- **混合精度训练**：AMP，加速并节省显存。
- **梯度累积**：模拟大 batchsize。

### 8.4 复现性

- 设置随机种子（Python、NumPy、PyTorch、CUDA）。
- 固定 cudnn 确定性：`torch.backends.cudnn.deterministic = True`。

### 8.5 常见问题解决

| 问题 | 解决方案 |
| :--- | :--- |
| 损失 NaN | 降低 lr，检查输入 NaN，梯度裁剪，换稳定激活 |
| 收敛慢 | 增加 lr，使用 BN，换 AdamW |
| 内存不足 | 减小 batchsize，使用梯度检查点，换小模型 |
| 类别不平衡 | 加权损失，过采样，Focal Loss，两阶段训练 |
| 小样本学习 | 迁移学习，元学习，强增强+正则化 |

## 9. 损失函数设计

| 任务 | 常用损失 | 变体与技巧 |
| :--- | :--- | :--- |
| 分类 | 交叉熵 | Label Smoothing；加权交叉熵 |
| 回归 | MSE / MAE | MSE 对异常值敏感，MAE 更鲁棒；Huber 损失 |
| 目标检测 | 分类 + 边界框回归 | Smooth L1, CIoU, GIoU；Focal Loss |
| 分割 | 交叉熵 + Dice Loss | Dice Loss 对小目标友好 |
| 生成对抗 | 对抗损失 + 感知损失 | LSGAN, WGAN-GP；感知损失用 VGG |
| 自监督 | 对比损失（InfoNCE） | 温度系数 τ=0.07~0.1 |
| 度量学习 | Triplet Loss | margin=0.2~1.0；hard negative mining |

**技巧**：
- 多任务学习：使用不确定性权重自动调整各损失权重。
- 不平衡回归：加权 MSE 或分位数损失。

## 10. 数据增强

### 10.1 常见增强操作

| 类型 | 操作 | 参数建议 |
| :--- | :--- | :--- |
| 几何变换 | 随机裁剪、翻转、旋转、缩放 | 翻转 p=0.5；旋转 ±15°；缩放 0.8~1.2 |
| 颜色变换 | 亮度/对比度/饱和度调整、灰度化 | 颜色抖动因子 0.2；灰度化 p=0.1 |
| 噪声 | 高斯噪声、椒盐噪声 | 标准差 0.01~0.05 |
| 擦除 | Random Erasing、Cutout | 擦除面积 0.02~0.4 |
| 高级 | MixUp、CutMix、RandAugment | 配合 AutoAugment 搜索 |

### 10.2 原则

- 增强强度与数据集规模成反比：小数据集用更强增强。
- 测试时不用增强（仅中心裁剪或尺寸归一化）。
- 序列数据（文本、时间序列）增强需谨慎（时间扭曲、加噪声）。

## 11. 快速决策速查表

### 11.1 Dropout

| 场景 | 类型 | 位置 | 比率 |
| :--- | :--- | :--- | :--- |
| 图像分类（全连接） | Standard | 全连接层后 | 0.5 |
| 自定义 CNN | SpatialDropout2D | 每个卷积块后 | 0.2 |
| 文本分类 | SpatialDropout1D | Embedding 后 | 0.2~0.3 |
| LSTM | Dropout + RecurrentDropout | 层间垂直 + 输出前 | 0.3~0.5 |
| Transformer | Standard | 每个子层后 | 0.1 |
| 数据量 <1000 | Standard | 仅最后一层前 | 0.5~0.7 |

### 11.2 激活函数

| 网络类型 | 隐藏层推荐 | 输出层（任务相关） | 顺序 |
| :--- | :--- | :--- | :--- |
| 普通 MLP/CNN | ReLU | 二分类: Sigmoid<br>多分类: Softmax<br>回归: 线性 | Linear/Conv → BN → ReLU |
| 深度 ResNet | ReLU | 同上 | Conv → BN → ReLU → (Add) → ReLU |
| Transformer | GELU | 同上 | Linear → GELU → Linear |
| LSTM | (内部 tanh/sigmoid) | 同上 | 额外 Linear 后接 ReLU |
| GAN 生成器 | LeakyReLU | Tanh | Conv → BN → LeakyReLU |
| 强化学习 | ReLU | 离散: Softmax<br>连续: Tanh/线性 | Linear → ReLU → Linear → 输出激活 |

### 11.3 归一化层

| 场景 | 推荐 Norm | 位置 | 备注 |
| :--- | :--- | :--- | :--- |
| 图像分类（batch≥16） | BatchNorm | Conv → BN → ReLU | 标准 |
| 图像分类（batch≤8） | GroupNorm | Conv → GN → ReLU | 或 LN |
| 目标检测/分割 | GroupNorm | 代替 BN | Mask R-CNN 已切换 |
| Transformer（NLP/ViT） | LayerNorm | Pre-LN: Norm → Attn → Add | 训练稳定 |
| RNN/LSTM | LayerNorm | 内部隐状态 | `layer_norm=True` |
| 风格迁移、GAN | InstanceNorm | Conv → IN → Activation | 保持内容结构 |
| 3D 卷积 | BatchNorm3D | 同 2D | 注意时序维度 |
| 自监督学习 | BatchNorm | 投影头中 | 大 batchsize |

### 11.4 优化器与学习率

| 场景 | 优化器 | 学习率调度 | 附加 |
| :--- | :--- | :--- | :--- |
| CNN（小数据） | SGD + Momentum | 阶梯下降 | lr=0.01, wd=1e-4 |
| Transformer | AdamW | 余弦退火 + 线性预热 | lr=3e-4, wd=0.01 |
| GAN | Adam | 固定或循环 | lr=2e-4, betas=(0.5,0.999) |
| 强化学习 | RMSprop / Adam | 固定 | lr=0.00025 |

### 11.5 训练调试速查

| 现象 | 可能原因 | 解决方案 |
| :--- | :--- | :--- |
| 训练损失不降 | 学习率太大/太小 | LR Finder，调整 lr |
| | 初始化不当 | 改用 He/Xavier |
| | 激活函数饱和 | 换 ReLU/LeakyReLU |
| 验证损失远高于训练 | 过拟合 | 增加 Dropout，L2，数据增强 |
| 损失震荡严重 | 学习率过大 | 降低 lr |
| | batchsize 太小 | 增大 batch 或使用 BN 替代 |
| 梯度范数极大 | 梯度爆炸 | 梯度裁剪，降低 lr |
| 梯度范数极小 | 梯度消失 | 换激活，调整初始化，加 BN |

---

## 总结

深度学习组件设计没有绝对唯一的标准答案，但遵循以下核心原则可大幅提高成功率：

1. **位置顺序**：`Linear/Conv → Norm → Activation → Dropout` 是大多数情况下的黄金组合。
2. **根据数据规模与任务选择正则化强度**：小数据多用 Dropout、L2、数据增强；大数据可适度减少。
3. **归一化类型取决于 batchsize 和网络结构**：大 batch 用 BN，小 batch 用 GN/LN，序列模型用 LN。
4. **优化器首选 AdamW + 余弦退火 + 预热**，分类/CNN 也可用 SGD + Momentum。
5. **训练前先过拟合一个 mini-batch**，确保模型实现正确；再逐步增加正则化。
6. **监控梯度范数、损失曲线**，快速定位问题。
7. **所有经验需结合具体任务 A/B 测试**，没有万能配方。
