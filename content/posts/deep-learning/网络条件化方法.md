在现代深度学习架构中，**条件化（Conditioning）** 是指让神经网络根据外部信息（如类别标签、文本描述、时间步长、风格图像等）动态调整其行为的能力。它是实现可控生成、多模态理解、风格迁移、视觉推理等前沿任务的核心技术。

本技术文档将对主流的条件化方法进行全面解析，涵盖其基本原理、数学模型、代码实现、应用场景与优缺点分析。

---

## 一、基础篇：特征融合方法

### 1.1 向量/特征拼接

**定义**：拼接是最直观的条件化方法，将表示条件的向量与主网络的特征向量在通道维度（或序列维度）上进行拼接，作为后续网络的输入。

**核心思想**：让网络学习条件信息与原始数据之间的交互关系。

**数学模型**：
$$
F_{\text{out}} = \text{Concat}(F_{\text{in}}, C)
$$
其中，$F_{\text{in}}$ 是原始特征，$C$ 是条件信息。拼接维度取决于任务需求：

- **通道拼接**：在卷积网络中，将条件向量沿通道维度与特征图拼接。
- **序列拼接**：在序列模型中，将条件标记（如 [CLS] token）拼接到输入序列中。

**示例代码**：

```python
import torch
import torch.nn as nn

class ConcatConditioning(nn.Module):
    def __init__(self, feature_dim, condition_dim):
        super().__init__()
        # 拼接后的特征通道数为原始特征 + 条件
        self.fc = nn.Linear(feature_dim + condition_dim, feature_dim)
        
    def forward(self, x, condition):
        # x: [batch, channels, height, width]
        # condition: [batch, cond_dim]
        batch, c, h, w = x.shape
        # 将条件广播到空间维度并与特征拼接
        cond_expanded = condition[:, :, None, None].expand(-1, -1, h, w)
        # 通道维度拼接
        x_concat = torch.cat([x, cond_expanded], dim=1)
        # 使用卷积或全连接层融合
        x_concat = x_concat.permute(0, 2, 3, 1).reshape(batch * h * w, -1)
        out = self.fc(x_concat).reshape(batch, h, w, -1).permute(0, 3, 1, 2)
        return out
```

**优点**：简单、直观、易于实现。

**缺点**：当条件信息与主特征的维度或空间结构不匹配时，需要额外的投影层；此外，拼接方法需要网络从零开始学习条件与数据之间的复杂交互。

**应用场景**：条件生成对抗网络（CGAN）、多模态特征融合、分类器条件化等。

---

### 1.2 特征相加

**定义**：将条件信息直接加到主网络的特征图上，要求两者具有完全相同的形状。

**核心思想**：通过加法实现条件信息的注入，是一种更轻量的特征融合方式。

**数学模型**：
$$
F_{\text{out}} = F_{\text{in}} + C
$$
其中，$F_{\text{in}}$ 和 $C$ 必须具有相同的形状。

**示例代码**：

```python
class AdditionConditioning(nn.Module):
    def __init__(self):
        super().__init__()
        
    def forward(self, x, condition):
        # x: [batch, channels, height, width]
        # condition: [batch, channels, height, width]（必须同形状）
        return x + condition
```

**优点**：实现极其简单，参数量为零，计算高效。

**缺点**：要求条件与主特征完全同形，限制了其应用范围；表达能力较弱。

**应用场景**：残差连接、位置编码注入、简单的条件引导等。

---

### 1.3 门控

**定义**：门控是一种更智能的特征融合方法，它利用条件信息计算一个门控向量，然后与主特征进行逐元素相乘，实现动态的特征通道开关。

**核心思想**：控制信息的流动，决定哪些特征被传递、哪些被抑制。

**数学模型**：
$$
F_{\text{out}} = G \odot F_{\text{in}}, \quad G = \sigma(\text{MLP}(C))
$$
其中，$\odot$ 表示逐元素相乘，$\sigma$ 是 Sigmoid 激活函数，$G \in [0,1]$ 是门控向量。

**示例代码**：

```python
class GatingConditioning(nn.Module):
    def __init__(self, feature_dim, condition_dim):
        super().__init__()
        self.gate_net = nn.Sequential(
            nn.Linear(condition_dim, feature_dim),
            nn.Sigmoid()
        )
        
    def forward(self, x, condition):
        # x: [batch, features, ...]
        # condition: [batch, cond_dim]
        gate = self.gate_net(condition)  # [batch, features]
        # 广播到所有空间维度
        gate = gate.unsqueeze(-1).unsqueeze(-1)
        return x * gate
```

**优点**：能够自适应地开关特征通道，比简单的加法和拼接更灵活；可解释性强（门控值直观反映了特征的重要性）。

**缺点**：额外的参数和计算开销。

**应用场景**：LSTM/GRU 中的遗忘门和输入门、多任务学习中的特征选择、动态网络剪枝等。

## 二、进阶篇：归一化调制方法

### 2.1 条件批归一化

**定义**：条件批归一化是标准批归一化的扩展，其缩放参数 $\gamma$ 和偏移参数 $\beta$ 不再通过反向传播学习得到固定的值，而是由条件信息动态生成。

**核心思想**：让 BN 层能够根据条件动态调整特征的分布，实现条件化的特征变换。

**数学模型**：
$$
\text{CBN}(x, c) = \gamma(c) \cdot \frac{x - \mu_{\mathcal{B}}}{\sqrt{\sigma^2_{\mathcal{B}} + \epsilon}} + \beta(c)
$$
其中，$\mu_{\mathcal{B}}$ 和 $\sigma^2_{\mathcal{B}}$ 是批次统计量，$\gamma(c)$ 和 $\beta(c)$ 是由条件 $c$ 预测得到的。

**示例代码**：

```python
class ConditionalBatchNorm2d(nn.Module):
    def __init__(self, num_features, condition_dim):
        super().__init__()
        self.num_features = num_features
        self.condition_dim = condition_dim
        
        # 使用条件信息预测 gamma 和 beta
        self.gamma_fc = nn.Linear(condition_dim, num_features)
        self.beta_fc = nn.Linear(condition_dim, num_features)
        
        # 可学习的 BN 参数（位置可保留或替换）
        self.eps = 1e-5
        
    def forward(self, x, condition):
        # x: [batch, channels, height, width]
        # condition: [batch, cond_dim]
        batch, c, h, w = x.shape
        
        # 计算批次统计量
        mean = x.mean(dim=(0, 2, 3), keepdim=True)
        var = x.var(dim=(0, 2, 3), keepdim=True, unbiased=False)
        
        # 根据条件生成 gamma 和 beta
        gamma = self.gamma_fc(condition).view(batch, c, 1, 1)
        beta = self.beta_fc(condition).view(batch, c, 1, 1)
        
        # 归一化后应用 gamma 和 beta
        x_norm = (x - mean) / torch.sqrt(var + self.eps)
        return x_norm * gamma + beta
```

**优点**：在生成对抗网络（GAN）中，CBN 能够解决使用标准 BN 导致的生成图片同质化问题；通过条件信息增强归一化层的表达能力。

**缺点**：批次统计量对小批量数据敏感，可能导致训练不稳定。

**应用场景**：条件图像生成、视觉问答（VQA）、多模态学习等。

### 2.2 空间自适应归一化

**定义**：SPADE 是条件归一化方法的一个重要突破，由英伟达在 GauGAN 中提出。与 CBN 不同，SPADE 的 $\gamma$ 和 $\beta$ 不再是全局的，而是**具有空间维度的**，可以根据空间位置动态生成。

**核心思想**：保留语义分割图中的空间布局信息，避免归一化层丢失重要的语义结构。

**数学模型**：
$$
\text{SPADE}(x, m) = \gamma(m) \cdot \frac{x - \mu}{\sigma + \epsilon} + \beta(m)
$$
其中，$m$ 是语义分割图（通常使用独热编码），$\gamma(m)$ 和 $\beta(m)$ 是通过卷积层从 $m$ 中学习得到的**空间自适应**参数。

**示例代码**：

```python
class SPADE(nn.Module):
    def __init__(self, num_features, label_nc):
        super().__init__()
        self.num_features = num_features
        self.param_free_norm = nn.InstanceNorm2d(num_features, affine=False)
        
        # 从语义分割图生成空间自适应的 gamma 和 beta
        nhidden = 128
        self.mlp_gamma = nn.Sequential(
            nn.Conv2d(label_nc, nhidden, 3, padding=1),
            nn.ReLU(),
            nn.Conv2d(nhidden, num_features, 3, padding=1)
        )
        self.mlp_beta = nn.Sequential(
            nn.Conv2d(label_nc, nhidden, 3, padding=1),
            nn.ReLU(),
            nn.Conv2d(nhidden, num_features, 3, padding=1)
        )
        
    def forward(self, x, segmap):
        # x: [batch, channels, height, width]
        # segmap: [batch, label_nc, height, width]（语义分割图）
        
        # 第一步：无仿射参数的归一化
        normalized = self.param_free_norm(x)
        
        # 第二步：从分割图生成 gamma 和 beta
        gamma = self.mlp_gamma(segmap)
        beta = self.mlp_beta(segmap)
        
        return normalized * gamma + beta
```

**优点**：解决了实例归一化在输入为统一语义标签（如全天空）时语义丢失的问题；能够根据语义布局**逐像素**调制特征，极大提升了语义图像合成的质量。

**缺点**：计算量较大；需要语义分割图作为条件输入。

**应用场景**：语义图像合成（GauGAN）、图像到图像的翻译、布局到图像生成等。

### 2.3 自适应实例归一化

**定义**：AdaIN 是风格迁移领域的革命性技术，它通过将内容特征的统计量（均值、方差）与风格特征对齐，实现了实时任意风格迁移。

**核心思想**：特征的均值和标准差承载了图像的风格信息，将内容特征的均值和标准差替换为风格特征的均值和标准差，即可实现风格迁移。

**数学模型**：
$$
\text{AdaIN}(x, y) = \sigma(y) \cdot \frac{x - \mu(x)}{\sigma(x)} + \mu(y)
$$
其中，$x$ 是内容特征，$y$ 是风格特征，$\mu$ 和 $\sigma$ 分别表示均值和标准差。

**示例代码**：

```python
class AdaIN(nn.Module):
    def __init__(self, epsilon=1e-5):
        super().__init__()
        self.epsilon = epsilon
        
    def forward(self, content, style):
        # content, style: [batch, channels, height, width]
        # 计算内容特征的统计量
        mean_content = content.mean(dim=(2, 3), keepdim=True)
        std_content = content.std(dim=(2, 3), keepdim=True)
        
        # 计算风格特征的统计量
        mean_style = style.mean(dim=(2, 3), keepdim=True)
        std_style = style.std(dim=(2, 3), keepdim=True)
        
        # 归一化内容特征，然后用风格的统计量重新缩放和平移
        normalized = (content - mean_content) / (std_content + self.epsilon)
        return normalized * std_style + mean_style
```

**优点**：实时任意风格迁移（从分钟级缩短到毫秒级）；不需要为每种风格重新训练模型；支持多风格插值。

**缺点**：仅使用全局统计量，可能丢失局部风格细节。

**应用场景**：实时艺术风格迁移、视频风格化、人脸属性编辑、域适应等。

### 2.4 自适应层归一化

**定义**：adaLN 是专门为扩散模型中的 Transformer 架构设计的条件化方法，其 $\gamma$ 和 $\beta$ 参数由时间步或文本嵌入等条件信息动态生成。

**核心思想**：将条件信息注入到 Layer Normalization 层中，实现对 Transformer 模块内部激活的调节。

**数学模型**：
$$
y = \frac{x - \mu}{\sigma + \epsilon} \cdot (1 + \gamma) + \beta, \quad \gamma = W_\gamma \cdot c, \quad \beta = W_\beta \cdot c
$$
其中，$c$ 是条件向量（如时间步嵌入），$W_\gamma$ 和 $W_\beta$ 是可学习的投影矩阵。

**示例代码**：

```python
class AdaptiveLayerNorm(nn.Module):
    def __init__(self, feature_dim, condition_dim):
        super().__init__()
        self.feature_dim = feature_dim
        self.norm = nn.LayerNorm(feature_dim, elementwise_affine=False)
        self.cond_gamma = nn.Linear(condition_dim, feature_dim)
        self.cond_beta = nn.Linear(condition_dim, feature_dim)
        
    def forward(self, x, condition):
        # x: [batch, seq_len, feature_dim]
        # condition: [batch, condition_dim]
        x_norm = self.norm(x)  # 先进行无仿射参数的层归一化
        gamma = self.cond_gamma(condition).unsqueeze(1)  # [batch, 1, feature_dim]
        beta = self.cond_beta(condition).unsqueeze(1)
        return x_norm * (1 + gamma) + beta
```

**adaLN-Zero 变体**：初始化生成 $\gamma$ 和 $\beta$ 的投影为零，使得模块在初始时表现为恒等函数，大幅提升了训练稳定性。

**优点**：参数高效（仅增加两个线性投影层）；训练稳定性好（特别是 Zero 变体）；在扩散 Transformer 中表现优异。

**缺点**：条件信息只能是全局的（无空间维度）。

**应用场景**：扩散 Transformer（DiT）、条件视频生成（CogVideo）、Stable Diffusion 等。

### 2.5 FiLM

**定义**：FiLM（特征级线性调制）是一种通用的条件化方法，它为中间特征图的**每个特征通道**生成 $\gamma$ 和 $\beta$ 参数，实现对特征的动态缩放和偏移。

**核心思想**：通过简单的仿射变换 $F_{\text{out}} = \gamma \odot F_{\text{in}} + \beta$ 实现对特征的灵活调制。

**数学模型**：
$$
\text{FiLM}(F, \gamma, \beta) = \gamma \odot F + \beta
$$
其中，$\gamma$ 和 $\beta$ 由条件信息通过 FiLM 生成器（通常是 RNN 或 MLP）生成。

**示例代码**：

```python
class FiLM(nn.Module):
    def __init__(self, feature_dim, condition_dim):
        super().__init__()
        # FiLM 生成器：从条件信息预测 gamma 和 beta
        self.film_generator = nn.Sequential(
            nn.Linear(condition_dim, 128),
            nn.ReLU(),
            nn.Linear(128, feature_dim * 2)  # 输出 gamma 和 beta
        )
        
    def forward(self, x, condition):
        # x: [batch, features, height, width]
        # condition: [batch, cond_dim]
        params = self.film_generator(condition)  # [batch, 2*features]
        gamma, beta = params.chunk(2, dim=1)  # [batch, features]
        
        # 广播到空间维度
        gamma = gamma.unsqueeze(-1).unsqueeze(-1)
        beta = beta.unsqueeze(-1).unsqueeze(-1)
        
        return x * gamma + beta
```

**优点**：通用性强，可作用于任意中间特征图；参数高效（FiLM 生成器的参数量远小于主网络）；在视觉推理任务中表现卓越（在 CLEVR 上将错误率减半至 97.7% 准确率）。

**应用场景**：视觉问答（VQA）、图神经网络（GNN-FiLM）、概率深度学习（FiLM-Ensemble）、异常检测等。

## 三、高级篇：权重生成方法

### 3.1 超网络

**定义**：超网络是一类特殊的神经网络，其输出不是普通结果，而是另一个神经网络（称为主网络或目标网络）的权重参数。

**核心思想**：超网络将条件信息映射到主网络的参数空间，实现对主网络行为的完全重塑。

**数学模型**：
$$
\theta_{\text{main}} = f_{\text{hyper}}(c)
$$
其中，$c$ 是条件向量，$f_{\text{hyper}}$ 是超网络，$\theta_{\text{main}}$ 是主网络的权重。

**示例代码**：

```python
class HyperNetwork(nn.Module):
    def __init__(self, input_dim, hidden_dim, target_weight_shape):
        super().__init__()
        # 超网络：条件输入 -> 目标网络的权重
        self.hyper = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, np.prod(target_weight_shape))
        )
        self.target_shape = target_weight_shape
        
    def forward(self, condition):
        # condition: [batch, cond_dim]
        weights = self.hyper(condition)  # [batch, total_params]
        weights = weights.view(self.target_shape)  # 重塑为目标网络的权重形状
        return weights
```

**优点**：极高的控制能力，能够根据条件完全重塑主网络的行为；参数高效（超网络本身参数量很小，通常低于 200MB）。

**缺点**：训练难度大；输出高维参数空间导致计算和存储开销大。

**应用场景**：生成模型微调（Stable Diffusion 的 Hypernetwork）、动态架构设计、实时语义分割（HyperSeg）等。

---

### 3.2 条件卷积

**定义**：条件卷积是一种动态卷积方法，它根据输入条件动态生成卷积核的权重，使卷积层能够针对不同样本自适应地调整其特征提取能力。

**核心思想**：每个卷积层拥有多套卷积核权重，条件函数根据输入动态决定各套权重的组合系数。

**数学模型**：
$$
W = \sum_{i=1}^{n} \alpha_i(c) \cdot W_i
$$
其中，$W_i$ 是第 $i$ 个专家卷积核，$\alpha_i(c)$ 是由条件 $c$ 预测的权重系数，$W$ 是最终的卷积核。

**示例代码**：

```python
class CondConv(nn.Module):
    def __init__(self, in_channels, out_channels, kernel_size, num_experts=8):
        super().__init__()
        self.num_experts = num_experts
        self.kernel_size = kernel_size
        self.in_channels = in_channels
        self.out_channels = out_channels
        
        # 专家卷积核集合
        self.weight = nn.Parameter(torch.randn(num_experts, out_channels, 
                                                in_channels, kernel_size, kernel_size))
        self.bias = nn.Parameter(torch.randn(num_experts, out_channels))
        
        # 条件路由网络：输入 -> 专家权重系数
        self.route_network = nn.Sequential(
            nn.AdaptiveAvgPool2d(1),
            nn.Flatten(),
            nn.Linear(in_channels, num_experts),
            nn.Softmax(dim=1)
        )
        
    def forward(self, x):
        # x: [batch, in_channels, height, width]
        # 第一步：根据输入预测专家权重系数
        route_weights = self.route_network(x)  # [batch, num_experts]
        
        # 第二步：组合专家卷积核
        kernel = torch.einsum('be, eoihw -> boihw', route_weights, self.weight)
        bias = torch.einsum('be, eo -> bo', route_weights, self.bias)
        
        # 第三步：执行卷积（每个样本使用不同的卷积核）
        # 注：标准卷积核形状为 [out, in, k, k]，需要调整
        batch, _, h, w = x.shape
        output = torch.zeros(batch, self.out_channels, h, w).to(x.device)
        for b in range(batch):
            output[b] = F.conv2d(x[b:b+1], kernel[b], bias[b:b+1],
                                 stride=1, padding=self.kernel_size//2)
        return output
```

**优点**：提升模型的容量与效率，在保持推理速度的同时提高精度；能够为每个样本学习特定的卷积核。

**缺点**：训练难度大；需要存储多个专家卷积核；推理时需要动态组合，可能增加计算复杂度。

**应用场景**：目标检测（CondInst）、图像分类（CondConv）、实时语义分割等。

## 四、其他主流方法

### 4.1 交叉注意力

**定义**：交叉注意力是 Transformer 架构中的核心机制，它在两个不同的序列之间建立依赖关系，利用一个模态的信息来增强另一个模态的表示。

**核心思想**：将一个序列作为 Query，另一个序列作为 Key 和 Value，通过注意力机制计算两者的相关性并加权融合。

**数学模型**：
$$
\text{CrossAttention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$
其中，$Q$ 来自模态 A（如文本），$K$ 和 $V$ 来自模态 B（如图像）。

**示例代码**：

```python
class CrossAttention(nn.Module):
    def __init__(self, dim_query, dim_key, dim_value, dim_output):
        super().__init__()
        self.query_proj = nn.Linear(dim_query, dim_output)
        self.key_proj = nn.Linear(dim_key, dim_output)
        self.value_proj = nn.Linear(dim_value, dim_output)
        self.out_proj = nn.Linear(dim_output, dim_output)
        self.scale = dim_output ** -0.5
        
    def forward(self, query, key, value):
        # query: [batch, query_len, dim_query]
        # key, value: [batch, key_len, dim_key/value]
        Q = self.query_proj(query)  # [batch, query_len, dim_output]
        K = self.key_proj(key)      # [batch, key_len, dim_output]
        V = self.value_proj(value)  # [batch, key_len, dim_output]
        
        # 计算注意力权重
        attn_scores = torch.matmul(Q, K.transpose(-2, -1)) * self.scale
        attn_weights = F.softmax(attn_scores, dim=-1)
        
        # 加权求和
        context = torch.matmul(attn_weights, V)  # [batch, query_len, dim_output]
        return self.out_proj(context), attn_weights
```

**优点**：能够捕捉序列间复杂的依赖关系；支持多模态信息的深层交互；是当前多模态模型的黄金标准。

**缺点**：计算复杂度为 $O(n^2)$，在处理长序列时开销较大。

**应用场景**：图像描述生成、视觉问答（VQA）、文本到图像生成（Stable Diffusion）、多模态 Transformer 等。

### 4.2 扩散模型的引导

#### 4.2.1 分类器引导

**定义**：分类器引导是一种在扩散模型采样过程中利用预训练分类器的梯度来引导生成方向的方法。

**核心思想**：在每一步去噪过程中，使用分类器的梯度调整预测噪声，使生成结果朝着目标类别方向偏移。

**数学模型**：
$$
\hat{\epsilon}_\theta(x_t, t) = \epsilon_\theta(x_t, t) - \sqrt{1 - \bar{\alpha}_t} \cdot s \cdot \nabla_{x_t} \log p_\phi(y | x_t)
$$
其中，$s$ 是引导强度，$\nabla_{x_t} \log p_\phi(y | x_t)$ 是分类器的梯度。

**示例代码**：

```python
def classifier_guidance_step(model, classifier, x_t, t, y_target, guidance_scale):
    # 预测噪声
    eps = model(x_t, t)
    
    # 计算分类器梯度
    x_t = x_t.requires_grad_(True)
    logits = classifier(x_t, t)
    loss = F.cross_entropy(logits, y_target)
    grad = torch.autograd.grad(loss, x_t)[0]
    
    # 引导噪声预测
    eps_guided = eps - guidance_scale * grad
    return eps_guided
```

**优点**：能够利用预训练的分类器实现零样本条件生成。

**缺点**：需要额外训练一个与时间相关的分类器；训练成本高；分类器的梯度可能不稳定。

**应用场景**：条件图像生成、可控扩散采样。

---

#### 4.2.2 无分类器引导

**定义**：无分类器引导（CFG）是目前扩散模型中最主流的引导技术，它通过混合条件和无条件模型的预测来实现高质量的条件生成，无需外部分类器。

**核心思想**：在训练时同时优化条件模型和无条件模型（通过随机丢弃条件），采样时通过外推条件模型预测与无条件模型预测之间的差异来增强条件影响。

**数学模型**：
$$
\hat{\epsilon}_\theta(x_t, t, c) = \epsilon_\theta(x_t, t) + w \cdot (\epsilon_\theta(x_t, t, c) - \epsilon_\theta(x_t, t))
$$
其中，$w$ 是引导权重，通常 $w > 1$ 时增强条件影响。

**示例代码**：

```python
def classifier_free_guidance_step(model, x_t, t, condition, cfg_scale):
    # 无条件预测（将条件设置为空）
    with torch.no_grad():
        eps_uncond = model(x_t, t, cond=None)
    
    # 有条件预测
    eps_cond = model(x_t, t, condition)
    
    # CFG 组合
    eps_guided = eps_uncond + cfg_scale * (eps_cond - eps_uncond)
    return eps_guided
```

**优点**：无需外部分类器；实现简单，效果好；是目前文生图模型的核心技术。

**缺点**：每次采样需要同时运行条件和无条件两次前向传播，计算开销加倍。

**应用场景**：Stable Diffusion、DALL·E 2、Imagen 等主流文生图模型。

---

### 4.3 生成对抗网络的条件化

**定义**：条件生成对抗网络（CGAN）是最早的条件化方法之一，它在生成器和判别器的输入中都加入条件标签，实现对生成过程的控制。

**核心思想**：通过将条件信息拼接到噪声向量和真实图像上，让生成器学习特定类别的生成，判别器学习判断图像是否与条件匹配。

**数学模型**：
$$
\mathcal{L}_{\text{CGAN}} = \mathbb{E}_{x \sim p_{\text{data}}(x)}[\log D(x|y)] + \mathbb{E}_{z \sim p_z(z)}[\log(1 - D(G(z|y)|y))]
$$
其中，$y$ 是条件标签。

**示例代码**：

```python
class ConditionalGenerator(nn.Module):
    def __init__(self, noise_dim, condition_dim, image_shape):
        super().__init__()
        self.fc = nn.Linear(noise_dim + condition_dim, 256)
        self.deconv = nn.Sequential(
            nn.ConvTranspose2d(16, 64, 4, 2, 1),
            nn.ReLU(),
            nn.ConvTranspose2d(64, image_shape[0], 4, 2, 1),
            nn.Tanh()
        )
        
    def forward(self, noise, condition):
        # 将噪声和条件拼接
        z = torch.cat([noise, condition], dim=1)
        features = self.fc(z).view(-1, 16, 4, 4)
        return self.deconv(features)
```

**优点**：实现简单直观；是最早也是最经典的条件化方法之一。

**缺点**：仅适用于离散标签条件；对复杂条件（如文本）支持有限。

**应用场景**：数字生成（MNIST 特定数字生成）、图像到图像翻译（Pix2Pix）、标签引导的图像生成。

## 五、总结与展望

| 方法 | 控制粒度 | 参数量 | 计算复杂度 | 核心优势 | 适用场景 |
|------|---------|--------|-----------|---------|---------|
| 拼接/相加 | 粗 | 低 | 低 | 简单直观 | 多模态融合 |
| 门控 | 中 | 中 | 中 | 自适应特征选择 | 特征开关、多任务学习 |
| CBN/SPADE | 细 | 中 | 中 | 空间自适应 | 语义图像合成 |
| AdaIN | 统计级 | 低 | 低 | 实时任意风格迁移 | 风格迁移、域适应 |
| adaLN | 通道级 | 低 | 低 | 扩散模型优化 | DiT、条件视频生成 |
| FiLM | 通道级 | 低 | 中 | 视觉推理性能卓越 | VQA、GNN、异常检测 |
| 交叉注意力 | 最细 | 高 | 高 | 多模态深层交互 | 文生图、多模态 Transformer |
| 超网络 | 完全 | 极高 | 高 | 完全控制 | 动态架构、模型微调 |
| 条件卷积 | 卷积级 | 高 | 中 | 动态特征提取 | 目标检测、高效推理 |
| CFG | 生成级 | 低 | 中（2倍） | 高质量可控生成 | 扩散模型文生图 |

选择合适的方法需要在**控制力**、**参数量**和**计算成本**之间进行权衡：

- **拼接和加法**最简单，但控制力有限。
- **归一化调制类方法**（CBN、SPADE、AdaIN、adaLN、FiLM）实现了控制力与参数量的平衡，是目前的主流选择。
- **交叉注意力**提供了最强大的多模态交互能力，是 Transformer 时代的核心范式。
- **超网络和条件卷积**提供了最高的控制上限，但参数量巨大，适用于对性能要求极高的场景。
- **CFG** 以计算开销换取高质量生成，已成为扩散模型的标配技术。
